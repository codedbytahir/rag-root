# Contributing to RAG ROOT 🚀

Thank you for your interest in contributing to RAG ROOT! We welcome contributions from everyone. Whether you're fixing a bug, adding a feature, or improving documentation, your help is appreciated.

## 🛣 Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/your-username/rag-root.git
   ```
3. **Create a new branch** for your work:
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/your-bug-fix
   ```

## 🛠 Development Workflow

1. **Install dependencies:**
   ```bash
   npm install
   ```
2. **Make your changes.** Ensure your code follows the project's style and conventions.
3. **Run linting:**
   ```bash
   npm run lint
   ```
4. **Test your changes.** Manually verify that your changes work as expected and don't break existing functionality.

## 📝 Commit Guidelines

- Use descriptive commit messages.
- Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
  - `feat: add new brain sharing feature`
  - `fix: resolve sidebar navigation glitch on mobile`
  - `docs: update API endpoint examples`

## 📬 Submitting a Pull Request

1. **Push your branch** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
2. **Open a Pull Request** against the `main` branch of the original repository.
3. **Describe your changes** clearly in the PR description. Include screenshots if applicable.
4. **Wait for review.** We will review your PR and provide feedback as soon as possible.

## 📜 Code of Conduct

Please be respectful and professional in all your interactions with the community.

## 💎 Design Guidelines

- We use a **3D glassy dark green** theme.
- Primary accent color: `#39E079`.
- Use Tailwind's `backdrop-blur` for overlays and modals.

Thank you for making RAG ROOT better!
